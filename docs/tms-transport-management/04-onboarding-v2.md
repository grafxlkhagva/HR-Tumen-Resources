# TMS Transport Management V2 — Хэрэглэгчийн гарын авлага

Шинэ хувилбарт нэмэгдсэн хамгийн том өөрчлөлтүүдийг нэгтгэж, хэрэглэгчдэд
шинэ боломжуудыг хурдан эзэмшихэд туслахаар бэлтгэв.

## 1. Нэг тээврийн удирдлагад олон үйлчилгээ

**Хуучин:** 1 гэрээ → 1 үйлчилгээ → олон машин.
**Шинэ:** 1 гэрээ → **олон** үйлчилгээ → үйлчилгээ тус бүрт олон машин.

### Хэрхэн үүсгэх вэ?

1. "Шинэ тээврийн удирдлага үүсгэх" дарж.
2. **Step 1** — "Гэрээт тээвэр".
3. **Step 2** — Гэрээгээ сонгосны дараа `checkbox` жагсаалтаас ядаж нэг үйлчилгээг сонго. Хэд сонгосон тооны badge "2 сонгогдсон" шиг гарна.
4. **Step 3** — Сонгосон үйлчилгээ бүрт өөрийн section. Section тус бүрд хамаарах машины "Бүгд" / "Цэвэрлэх" товчнууд.
5. **Үүсгэх** → detail хуудас руу ирнэ. Dispatch, санхүү, chung status нь бүх дэд тээврүүдээр нэгтгэгдсэн байна.

> Жишээ: Ачааны тээвэр + түрээсийн тээвэр нэг TM дотор хянагдана.

## 2. Харилцагчийн үнэ + автомат ашиг

Финансын карт:
- **Өртөг / Жолоочийн үнэ** — гараар.
- **Харилцагчид өгч буй үнэ (НӨАТ-гүй)** — **гараар**.
- **Ашгийн хувь** — автомат (читайлах `(customer − driver) / driver × 100`).

Хэрэв харилцагчийн үнэ жолоочийн үнээс бага байвал "Харилцагчийн үнэ жолоочийн үнээс бага байна" анхааруулах баталгаа гарна. Батлавал **улаан өнгөөр алдагдалтай гэж тэмдэглэгдэнэ**.

## 3. Машин сонгосны дараа жолоочийн шүүлт

- Тээврийн хэрэгсэл засах → Машин сонгоно.
- Ялгавар: Жолоочийн dropdown тэр машинд **оноосон** жолоочдоор автоматаар шүүгдэнэ.
- Ганцхан жолоочтой бол автомат оноо. Олонтой бол дарж сонгоно.
- Машинд оноосон жолооч байхгүй бол бүх жолооч харагдана (fallback).

## 4. Concurrent edit хамгаалалт

Хоёр хүн зэрэг нэг TM засахад:
- Хамгийн түрүүн хадгалсан нь ноёрхоно.
- Хоёрдугаар хэрэглэгчид "Зөрчил илэрлээ — Хуудсыг дахин ачааллана уу." toast гарна.
- Гараар refresh хийж дахин засна.

## 5. Жагсаалт дээрх нэмэлт мэдээлэл

- Multi-service / multi-machine TM-д `"N үйлчилгээ · M машин"` гэсэн жижиг badge.

## 6. Хэвийн бус нөхцөлүүд

| Тохиолдол | Зан төлөв |
|---|---|
| Гэрээнд машин бүртгэгдээгүй үйлчилгээ | 1 машингүй дэд таб үүсч, машиныг дараа оноодог. |
| Хуучин TM (single-service) | Өөрчлөлтгүй ажиллана. |
| Хасах маржинтай хадгалахыг оролдох | Confirm dialog гарна. |
| Хоёр үйлчилгээний сервистэй машин | Машиныг хоёр subUnit-д давхар сонгоно (үйлчилгээ өөр учраас утгатай). |
